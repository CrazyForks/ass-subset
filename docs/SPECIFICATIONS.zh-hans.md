## 绘图指令无损还原规范

**支持版本**：v2.7+

当绘图指令转换为字体字形时，原始指令信息需要被保存，以便日后还原。本工具在**绘图指令字体**中创建自定义的 `draw` 表来记录这些信息。

### 字形分配规则

绘图指令通过转换为字体中的矢量轮廓（字形）来实现嵌入。按**字幕时间顺序**，每个绘图指令被转换为一个字形，并分配一个对应的字符作为引用：第一个出现的指令转换为字形 `A`，第二个转换为字形 `B`，以此类推。字符分配过程跳过 ASS 保留符号（`{`、`}`、`\`、`-`）和不可见字符（空格、制表符等）。当字幕中的绘图指令被删除或修改时，`draw` 表会自动重排，后续字形的分配字符向前递进，字幕中的所有引用也同步更新。这一字形分配规则自本工具最初版本起就被采用，绘图指令字体初期命名为 `ASSDrawSubset_MontageSubs`，之后统一改名为 **`ASSDrawSubset`** 以实现标准化。

### 数据存储规范

自定义 `draw` 表用于存储所有转换的绘图指令与其对应字形的映射关系。表的结构如下：

#### 表头结构

表的开头是一个 **32 位无符号整数（大端序）**，记录表中条目的总数。例如，如果转换了 15 个不同的绘图指令，此值为 15。

#### 条目结构

表头之后，每个条目按顺序排列。每个条目包含以下三个字段：

| 字段名 | 类型与长度 | 含义 | 示例 |
|--------|-----------|------|------|
| **data** | uint16 长度 + UTF-8 字符串 | 原始绘图指令完整文本 | `m 100 100 l 200 200 c 250 250 300 300 350 350` |
| **char** | uint8 长度 + UTF-8 字符串 | 对应字形的单个 Unicode 字符 | `A` 或 `①` 或任意未使用字符 |
| **flags** | 1 字节位字段 | 精度等级与结束标记 | 见下表 |

#### flags 字段规范

flags 字节的各位含义如下：

| 位范围 | 含义 | 取值范围 | 说明 |
|--------|------|--------|------|
| 位 0-3 | 绘图精度等级 | 1 或 2 | 值为 1 表示原指令精度为 `\p1`；值为 2 表示精度为 `\p2` |
| 位 4 | \p0 结尾标记 | 0 或 1 | 值为 1 表示原指令在末尾包含 `\p0`；值为 0 表示无 |
| 位 5-7 | 保留位 | 0 | 目前未使用，必须为 0，未来可能扩展 |

**示例**：原指令为 `\p2 m 100 100 l 200 200 \p0`，则 flags 值为 `0b00010010` = 18（十进制）。其中位 0-3 = 2（精度 \p2），位 4 = 1（有 \p0），位 5-7 = 0。

### 还原流程

当用户在工具中点击"移除内嵌字体"时，工具执行以下步骤：

1. **扫描绘图指令字体** — 遍历字幕中的所有嵌入字体，仅关注包含 `draw` 表的字体
2. **检查 draw 表** — 确认字体是否包含 `draw` 表；若不含则跳过该字体
3. **读取映射关系** — 按顺序读取表头的条目数，然后逐条读取每个条目的 `data`、`char` 和 `flags` 字段
4. **还原指令** — 在字幕的绘图指令区域内查找该字体的所有字形引用；根据 `flags` 字段恢复 `\p` 精度标记和 `\p0` 结尾
5. **删除字体文件** — 还原完成后，从字幕中删除已还原的字体文件

**还原示例**：若原字形 `A` 对应指令 `m 100 100 l 200 200`、精度为 `\p2`、有 \p0，则恢复为 `\p2 m 100 100 l 200 200 \p0`。

### 验证与调试

提供命令行工具 [`/scripts/python/draw_font_inspector.py`](/scripts/python/draw_font_inspector.py) 用于检查绘图指令字体内的 `draw` 表：

- 列出所有映射条目及其内容
- 显示每个条目的 `data`、`char` 和 `flags` 值
- 验证 flags 字段的位组成是否符合规范
- 辅助开发者调试绘图指令转换过程

**规范维护说明**：v2.7+ 的 `draw` 表规范为当前版本。未来可能根据实际需求在 flags 字段中增加更多位标记，或扩展表结构。规范更新时，本文档和诊断脚本将同步更新。

## 字体随机化命名与映射还原规范

**支持版本**：v2.6+

### 问题背景与字幕社区解决方案

#### 播放器缓存问题

字体子集化是应对中日韩字体字符过多的常见优化方案。然而，当字幕文件跨集合或单个 MKV 中包含多集内容时，会暴露出播放器缓存问题：

**问题描述**：播放器会缓存第一次加载的字体及其已支持的字符集。由于子集化后的字体只包含必要字符，当播放到第二集使用同名字体但字符集不同时，播放器不会重新加载字体，导致缺失字符无法正常显示。

#### 字幕社区的解决方案：随机化命名

为解决此问题，中日韩字幕社区创新性地提出了**字体随机化命名**方案：每次嵌入字体时，都为其分配一个随机的名称，强制播放器将其视为新字体并重新加载，从而避免缓存问题。

### 随机化命名规则

#### 命名生成

每当字体被嵌入时，工具为其生成一个随机的 8 字符名称，用于替代原始字体名称。随机名称由以下字符组成：

- 大写字母：A-Z（26 个）
- 数字：0-9（10 个）
- **总组合数**：36^8，足以保证随机性

**示例**：Arial 字体可能被随机化为 `QTSGQJIO`、`8X2KL5PN` 等。

#### 应用范围

随机化名称应用在字幕的以下所有位置：

- 样式表中的字体名称声明
- 对话行中的字体标签（如 `{\fnQTSGQJIO}`）
- 嵌入字体文件本身的名称表

这样，每次处理字幕时都会生成不同的随机名称，强制播放器重新加载字体而非使用缓存。

### 映射记录规范

为了能够还原原始字体名称，随机化名称与原始名称的映射关系通过两种方式记录，形成双重备份机制。

#### 方式 1：ASS 字幕脚本信息注释（传统方案）

这是中日韩字幕社区开发的子集化工具的通行做法。在 ASS 字幕文件的 `[Script Info]` 部分添加注释，格式为：

```
; Font Subset: 随机名称 - 原始名称
```

**示例**：
```
[Script Info]
; Font Subset: QTSGQJIO - Arial
; Font Subset: 8X2KL5PN - SimHei
ScriptType: v4.00+
PlayDepth: 10
...
```

**优缺点**：易于人类阅读和手动编辑，但若用户用不规范的文本编辑器修改字幕，这些注释行可能会被意外删除。

#### 方式 2：字体文件 ID 10 表备份（本工具增强方案）

在本工具 v2.6+ 版本中，我们额外在**嵌入字体文件的 ID 10 表**（Description）中保存映射数据作为备份。即便字幕注释被误删，映射信息仍然存在于字体文件本身。

在字体的 ID 10 表中，添加 YAML 格式的元数据 `FontSubsetMap`，包含以下字段：

| 字段 | 类型 | 含义 | 示例 |
|------|------|------|------|
| **original** | 字符串 | 原始字体的真实名称 | `Arial` |
| **subset** | 字符串 | 随机化后的名称 | `QTSGQJIO` |
| **ass-subset** | 版本字符串 | 生成此映射的工具版本 | `2.6` 或 `2.7` |

**ID 10 表完整记录示例**：
```
FontSubsetMap: {original: Arial, subset: QTSGQJIO, ass-subset: 2.6}
```

### 还原流程

用户在工具中点击"移除内嵌字体"时，工具执行以下步骤：

#### 第 1 步：检查字体映射备份

工具遍历字幕中的每个嵌入字体，尝试读取其 ID 10 表。如果表中包含有效的 `FontSubsetMap` 元数据，提取其中的 `original` 和 `subset` 字段，作为映射关系。

#### 第 2 步：兼容性查询

如果某个字体的 ID 10 表中不存在 `FontSubsetMap`，工具尝试在字幕的 `[Script Info]` 部分查找对应的 `; Font Subset:` 注释，获取备用映射。

#### 第 3 步：替换字幕中的字体名称

对于找到映射关系的字体，工具在字幕的所有位置搜索该随机化名称（`subset` 字段），并全部替换为原始名称（`original` 字段）。替换范围包括：

- 样式表中的字体声明行
- 对话行中的字体标签
- 其他任何文本位置

**替换示例**：若映射为 `original: Arial, subset: QTSGQJIO`，则所有 `{\fnQTSGQJIO}` 替换为 `{\fnArial}`，所有独立出现的 `QTSGQJIO` 也替换为 `Arial`。

#### 第 4 步：删除嵌入字体

替换完成后，从字幕中移除所有嵌入的字体文件。播放器在播放时会使用系统安装的原始字体进行渲染。

#### 映射优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 第 1 优先 | 字体 ID 10 表 | 更可靠，不易被意外删除 |
| 第 2 优先 | ASS 注释 | 备用方案 |
| 无映射 | 两处均无 | 不进行任何操作，并提示用户该字体无法移除 |

### 验证与调试

提供命令行工具 [`/scripts/python/font_name_inspector.py`](/scripts/python/font_name_inspector.py) 用于检查字体 name 表。
